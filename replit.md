# Telegram Signals Backtester

A React + Express application for backtesting Telegram trading signals against real historical market data (1-minute and 60-minute OHLCV candles), with exact pip/spread/slippage/position-sizing mathematics and an MT5 Expert Advisor export pipeline.

---

## Quick Start (Replit or Google AI Studio)

```bash
npm install       # install all dependencies (only needed once or after fresh upload)
npm run dev       # starts Express + Vite on port 5000
```

The `SESSION_SECRET` Replit secret must be set — Replit auto-generates it. All other secrets are optional.

## Workflow

The "Start application" workflow runs `./node_modules/.bin/tsx server.ts` on port 5000. The server:
- Binds `0.0.0.0` and reads `process.env.PORT` (defaults to 5000)
- Serves the Vite dev server in development (HMR included)
- `vite.config.ts` sets `server.allowedHosts: true` — required for Replit's proxied preview

## Market Data Architecture

`fetch1mCandlesFromApi` in `server.ts` tries providers in order (8-second timeout each):

```
DataSource = "custom" | "polygon" | "twelvedata" | "oanda" | "yahoo" | "yahoo_1h" | "synthetic"
```

- **Tiers 1–5** return 1-minute candles; **Tier 6** (Yahoo 60m) covers ~730 days; **Tier 7** (Synthetic) is Brownian-motion, reproducible by seed.
- Only real (non-synthetic) candles are persisted to the SQLite cache. A cache hit always means real data.
- `SimulatedTrade.isRealData` is derived from `dataSource !== "synthetic"` — never hardcoded.

## Telegram Signal Filtering

When fetching messages from Telegram channels, the server automatically:
- **Excludes** trade result/status messages: "TP1 hit", "running 80 pips", "BE hit", "trade closed", etc.
- **Flags** forwarded messages (`isForwarded: true`, `forwardedFrom: "Channel Name"`) so the AI parser can deprioritize short forwarded snippets.
- **Applies** a ticker + action + level gate to reduce AI token usage before sending to Gemini/Groq.

## Backtest Engine

`backtestSignalWithRealData` in `server.ts`:
- Two precision modes: `"candle"` (SL always wins same-bar conflicts) or `"every_tick"` (OHLC wick-dominance model)
- `DataSource` is threaded from fetch → trade result — never hardcoded as "real"
- Sim console shows `⚡ REAL DATA [YAHOO_1H]` or `🎲 SYNTHETIC — no real data available` per trade

---

## Optimizer

The **600-Combination Grid Search** (`POST /api/optimize`) tests:
- 4 TP/SL exit configs × 3 trailing configs × 10 lot strategies × 5 risk levels
- Walk-Forward IS/OOS split (70%/30%) with overfit risk detection
- Score = `Sharpe_IS × 30 + PF_OOS × 25 − DrawdownPenalty`
- Errors are surfaced in the UI — no silent failures

---

## MT5 Export Pipeline

**CSV** (`/api/signals/export-csv`): 7-column format — `Timestamp,Symbol,Type,Price,SL,TP,LotSize`
- Timestamps always UTC (`getUTC*` methods)
- Missing datetime → `"1970.01.01 00:00:01"` (fires on first MT5 bar)

**EA** (`public/SignalReplayEngine.mq5`): reads all 7 columns, handles broker symbol suffixes.
